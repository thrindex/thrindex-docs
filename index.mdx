## From Python to hardware in three steps

```bash
pip install thrindex
```

```python
import thrindex as thx
import thrindex.snn as snn

# 1. Author
model = snn.Sequential(
    snn.Dense(700, 512),
    snn.LIF(tau_mem=20.0, threshold=0.3),
    snn.Dense(512, 20),
    snn.LIF(tau_mem=20.0, threshold=0.3),
)

# 2. Compile — resolves tau_mem → alpha, seals with CRC32
thx.compile(model, "model.thx")
```

```bash
# 3. Run — simulator or hardware, same command
thrindex run model.thx
```

## THRINDEX Certified

Every hardware backend earns its badge by passing the conformance harness against `CONFORMANCE_ENVELOPE_V0` — a precisely derived bound on how far a backend's outputs can diverge from the float reference simulator.

| Metric | Threshold | Meaning |
|---|---|---|
| `T_mean` | ≤ 0.020 | Mean spike-count error ≤ 2% |
| `T_max` (p99) | ≤ 0.130 | Worst-neuron error ≤ 13% |
| `P_pred` | ≥ 0.900 | ≥ 90% of predictions agree |

The reference simulator passes with T_mean = 0, T_max = 0, P_pred = 1.0. The badge is a contract, not a claim.

## What THRINDEX is not

- **Not a general-purpose framework.** It targets spiking neural networks on neuromorphic hardware specifically.
- **Not a black box.** The output of every run is a structured transcript — spike counts, syn-ops, energy estimate — you can inspect, diff, and test.
- **Not an abstraction over hardware.** It gives you a deterministic model of what will happen on the target, and a conformance bound on the divergence.
